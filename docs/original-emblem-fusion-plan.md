# Original Emblem / Fusion Migration Plan

`dungeon-rpg`에 **원본형 합체 시스템**을 이식하기 위한 기준 문서.

> 상태: draft
> 목표: 축약형 전투 합체가 아니라, `lightsaber_test` 문서에 정리된 **문장(Emblem) / 문장의방 / 8~10단 융합 구조**를 가능한 한 원본 감각대로 가져온다.

---

## 1. 전제

이번 작업에서 말하는 "합체 시스템"은 아래를 의미한다.

- 동료 둘을 전투 중 잠깐 합체하는 액션 시스템이 아님
- 현재 `dungeon-rpg`의 시너지 상위판도 아님
- **문장 획득과 상위 전직 구조가 묶인 장기 성장 시스템**임

즉 원본 기준 합체는:

1. 7단 Lv36 도달
2. 문장의방 진입
3. 병종별 기본 문장 획득
4. 라인별 문장 합체
5. 8단 마스터 직업 진입
6. 8단 Lv100 → 9단 그랑
7. 9단 Lv200 → 10단 그랜드

의 구조를 가진다.

---

## 2. 현재 `dungeon-rpg`와의 차이

현재 `dungeon-rpg`는 아래 상태다.

- 플레이어 성장 라인: 사실상 `infantry` 1개 중심
- 승급: 현재 프로토타입은 Lv35 cap 기반 경량 승급이었음 (정본 동기화 대상)
- 장비 스탯: 단순화 버전
- 동료: 10병종 roster지만 동료 수집/전직/문장 구조는 없음
- 합체: 미구현
- 문장의방: 미구현
- 8~10단 장기 레벨 구조: 미구현

즉 지금 구조 위에 합체 버튼 하나 올리는 걸로는 **원본 구현이라고 볼 수 없다.**

---

## 3. 원본형 구현에 반드시 필요한 축

원본 기준으로 가려면 최소 아래 6축이 같이 움직여야 한다.

### 3.1 클래스/단수 구조 확장
- 현재의 단순 `classRank`를 넘어
- **1단~10단 전체 승급 구조** 필요
- 3단 이후 전직 분기
- 8단 이상은 융합 클래스

### 3.2 병종 라인 전체화
- 보병만이 아니라
- 비병 / 기병 / 수병 / 창병 / 궁병 / 승려 / 신관 / 법사 / 사교
- 최소한 데이터 구조상 전부 존재해야 함

### 3.3 문장 시스템
- 기본 문장 10종
- 마스터 문장 3종
- 캐릭터별 보유 문장 상태
- 문장 보너스 스탯 계산

### 3.4 문장의방
- 진입 조건 판정
  - tier 7+
  - level 36+
  - 병종 일치
  - 공격/방어 조건 충족
- 시험 전투 또는 대체 판정 흐름
- 문장 획득 처리

### 3.5 문장 합체
- 재료 문장 전부 보유 시 합체 가능
- 재료 제거
- 마스터 문장 지급
- 상위 직업 진입 조건 해금

### 3.6 8~10단 장기 성장
- 8단 max level 100
- 9단 max level 200
- 10단 max level 300
- 상위 융합 클래스 데이터
- 레벨 상한/경험치 커브 확대

## 4. 원본 기준 합체 계열

문서 기준 고정 라인은 다음으로 본다.

### BattleMaster
- 보병
- 비병
- 기병

### TacticsMaster
- 수병
- 창병
- 궁병

### MagicMaster
- 승려
- 신관
- 법사
- 사교

합체 결과는:
- 8단 마스터
- 9단 그랑
- 10단 그랜드

로 이어진다.

---

## 5. `dungeon-rpg`에 맞는 현실적 이식 원칙

원본형으로 가더라도, 아래는 먼저 결정해야 한다.

### 5.1 무엇을 먼저 원본화할 것인가
원본 전체를 한 번에 다 넣으면 위험하다.

그래서 실제 구현 순서는 아래처럼 잘라야 한다.

#### Phase O1. 데이터 구조 원본화
- 10병종 player line 데이터 추가
- 1~10단 tier/rank 테이블 재정의
- emblem data 테이블 추가
- fusion recipe 테이블 추가

#### Phase O2. 문장 획득 루프
- 문장의방 패널 추가
- 진입 조건 검사
- 기본 문장 획득 상태 저장
- 플레이어/캐릭터 스탯에 문장 보너스 반영

#### Phase O3. 합체 및 8단 진입
- 마스터 문장 합체 UI
- 재료 소모
- 마스터 문장 지급
- 8단 클래스 진입 처리

#### Phase O4. 9단/10단 장기 확장
- 8단~10단 레벨 상한 확장
- 그랑/그랜드 클래스 데이터 추가
- 계승 문장 또는 상위 해금 규칙 추가

### 5.2 현재 시스템과 충돌하는 것
원본형으로 가면 아래 일부는 다시 뜯어야 한다.

- 현재의 `infantry` 단일 라인 가정
- 현재 문서 기준 canonical 값은 7단 Lv36
- 현재 경량 `classRank` 표시 방식
- 현재 수련의 방 중심 승급 해석

즉 현재 구현은 버릴 필요까진 없지만,
**원본형 기반으로 재해석되도록 리모델링**해야 한다.

---

## 6. 추천 구현 순서, 강한 권장

원본형이라고 해도 나는 이 순서가 맞다고 본다.

### 1단계, 데이터/세이브 뼈대
가장 먼저 할 것:
- player growth line 구조 일반화
- emblem inventory/save 구조 추가
- fusion class metadata 추가
- tier max level 테이블 확장

이게 먼저 없으면 UI부터 만들어도 다 다시 고쳐야 함.

### 2단계, 문장의방
그 다음:
- 문장의방 NPC/시설 추가
- 진입 조건 판정 UI
- 기본 문장 획득 플로우

### 3단계, 문장 합체
그 다음:
- 합체 메뉴
- 합체 가능 여부 표시
- 합체 결과 문장 지급
- 8단 진입 가능화

### 4단계, 상위 전직/장기 커브
마지막:
- 8단/9단/10단 진입
- 레벨 캡 100/200/300
- 장기 XP 곡선
- 상위 클래스명/표시/UI 정리

---

## 7. 구현 전에 고정해야 할 결정

원본을 그대로 옮기려면 아래를 고정해야 한다.

### 7.1 플레이어만 적용할지, 동료까지 적용할지
이게 가장 큼.

선택지:
1. **플레이어만 원본형 문장/융합 적용**
2. 플레이어 + 동료 전부 적용

원본 감각은 2번에 가깝지만,
`dungeon-rpg` 프로토타입 현실성은 1번이 훨씬 안전하다.

그래도 사용자가 "원본 그대로"를 강하게 원하면,
최소 문서/데이터 구조는 2번 확장 가능하게 잡아야 한다.

### 7.2 시험 전투를 실제로 넣을지
문장의방은 문서상 시험 전투가 있음.

선택지:
- A. 실제 1vs1 시험 전투 구현
- B. 조건 충족 시 즉시 획득

원본형이면 A가 더 맞다.

### 7.3 3단 이후 분기를 얼마나 넓게 복구할지
합체를 원본형으로 하려면 사실 3단 이후 클래스 분기도 어느 정도 복원돼야 한다.

왜냐면 7단 라인 최고 단계가 존재해야 문장 구조가 자연스럽기 때문.

---

## 8. 지금 바로 시작할 첫 구현 단위

원본형으로 정말 갈 거면,
가장 먼저 건드릴 파일은 대체로 이쪽이다.

- `js/data.js`
  - tier table
  - player line table
  - emblem definitions
  - fusion recipes
- `js/state.js`
  - player emblem state
  - currentClassId / classHistory에 준하는 구조
- `js/save.js`
  - emblem/fusion persistence
- `js/ui-panels.js`
  - 문장의방 패널
  - 문장 목록 / 합체 메뉴
- `js/transitions.js` / `js/main.js`
  - 문장의방 진입 흐름
- `index.html`
  - 문장의방 패널 마크업

즉 첫 작업은 거의 확실하게
**데이터 구조 + 세이브 구조 재설계**다.

---

## 9. 결론

사용자가 원하는 "원본 버전 그대로"는,
현재 프로토타입에 붙일 수 있는 가벼운 합체 기능이 아니라

**성장 시스템 후반부 전체를 원본 기준으로 재편하는 프로젝트**다.

그래서 다음 액션은 이것이다.

1. 축소형 합체안 폐기
2. 원본형 문장/융합 기준으로 문서/데이터 구조 고정
3. 데이터/세이브 뼈대부터 구현 시작

이 방향이면 느리더라도 맞는 방향으로 간다.
